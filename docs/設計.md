# PrefxPy 設計メモ（MVP）

このドキュメントは、実装を小さく保ちつつ将来の変更をしやすくするための設計メモです。構成はリリース単位で見直してよい。

## 方針

- まずは最小機能で公開する。
- 機能の境界を分け、将来の差し替えを容易にする。
- テストを先に充実させ、仕様変更に耐える。

## MVP スコープ（v0.1.x）

1. HTTP Reverse Proxy
2. Prefix 配下公開（例: `/myapp/`）
3. Forwarded ヘッダ付与
4. Redirect `Location` の prefix 補正
5. `prefxpy.toml` による起動設定（v0.1.1）
6. 設定からの追加ヘッダ（Forwarded 系は常に上書き）（v0.1.1）

## モジュール構成

- `prefxpy.cli`
  - 引数の解決、`web.run_app` への委譲のみ
  - `resolve_settings` で CLI と TOML をマージし、テストから直接呼べるようにする
- `prefxpy.config`
  - TOML の読み込み（3.11+: `tomllib`、3.10: `tomli`）
  - `[headers]` の検証と不変のタプルへ正規化
- `prefxpy.proxy`
  - ルーティング、prefix 処理、上流への転送、ヘッダ処理

## CI / 公開

- `.github/workflows/ci.yml` … `pytest` を Python 3.10–3.12 で実行
- `.github/workflows/publish.yml` … タグ `v*` で本番 PyPI、手動で TestPyPI／本番を選択可能
- 手順の詳細: [`docs/PYPI.md`](PYPI.md)

## 将来の拡張

- `prefxpy.rewrite`: HTML/JS の rewrite 機能
- `prefxpy.scan`: 静的検査・警告出力
- WebSocket 中継

## メンテナンス規約（軽量）

- 変更は小さな差分で行う。
- 非目標（本番プロキシ化）は README に常に明記する。
- 破壊的変更は 0.x の間でも CHANGELOG に理由を書く。
- 依存パッケージを追加・変更したら [`THIRD_PARTY_NOTICES.md`](../THIRD_PARTY_NOTICES.md) を更新する。
