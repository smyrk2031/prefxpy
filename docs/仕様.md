# PrefxPy 仕様（実装反映）

バージョンごとの差分は [`CHANGELOG.md`](../CHANGELOG.md) を参照。

## CLI

```bash
prefxpy --target http://localhost:8000 --prefix /myapp/ --host 127.0.0.1 --port 9000
```

引数なしの `prefxpy` でも起動可（`target` 既定 `http://127.0.0.1:8000`）。

### オプション

- `--config` 任意。TOML 設定ファイルへパス（既定のキーは `--target` 無しでも起動可）
- `--target` 上流アプリ URL（未指定時は設定ファイル、なければ `http://127.0.0.1:8000`）
- `--prefix` 公開prefix（未指定時は設定ファイル、なければ `/myapp/`）
- `--host` bind host（未指定時は設定ファイル、なければ `127.0.0.1`）
- `--port` bind port（未指定時は設定ファイル、なければ `9000`）
- `--log-level` ログレベル（`DEBUG|INFO|WARNING|ERROR`、未指定時は設定ファイル、なければ `INFO`）

**優先順位:** CLI の引数は、設定ファイルにある同じキーより優先する。

## `prefxpy.toml`（設定ファイル）

キー例（値は任意）:

- `target`（文字列）必須※ ※CLI で `--target` がある場合は省略可
- `prefix`（文字列）既定 `/myapp/`
- `host`（文字列）既定 `127.0.0.1`
- `port`（整数）既定 `9000`
- `log_level`（文字列）既定 `INFO`
- `[headers]` テーブル: 上流へ付与する追加ヘッダ（値は文字列化）。`X-Forwarded-*` はプロキシ側の正規値が常に優先され上書きされる

サンプル: [`examples/prefxpy.example.toml`](../examples/prefxpy.example.toml)

## 動作

1. `/` は案内 HTML を返す（アプリ本体ではない）。
2. `prefix` で始まるパスだけ上流へ転送する。それ以外は 404 と案内文。
3. 受信時に `prefix` を剥がして上流へ転送する。
4. `X-Forwarded-*` ヘッダを付与する（`[headers]` と重複する同名キーは上書き）。
5. 上流レスポンスの `Location: /...` は `Location: {prefix}/...` に補正する。
6. 起動時に Public URL をコンソールへ表示する。
7. `/_prefx/health` はヘルスチェック用固定エンドポイント。

- 本番向け高性能プロキシ
- HTML/JS 自動 rewrite
- WebSocket 中継
