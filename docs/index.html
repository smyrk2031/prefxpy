PrefxPy

A tiny local reverse proxy to reproduce production prefix and forwarded-header behavior.

Install 
pip install prefxpy
Run 
prefxpy

Opens http://127.0.0.1:9000/myapp/ and forwards to http://127.0.0.1:8000 by default.

Why use it?
  • Apps that break only under a subpath like /myapp/
  • Missing or wrong X-Forwarded-Prefix handling
  • Local checks before deploying behind nginx or a load balancer
